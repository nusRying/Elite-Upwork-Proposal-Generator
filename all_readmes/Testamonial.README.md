# Testamonial Website

Static wedding invitation site ready for GitHub Pages hosting.

## Live URL

After deployment, your site will be available at:

`https://nusrying.github.io/Testamonial/`

## Privacy and security notes

- This is a public GitHub Pages site. Never place private secrets (database passwords, service-role keys, API private keys) in `index.html`, `main.js`, or any committed file.
- Use only public-safe keys in frontend code (for example: Supabase `anon` key), and keep strict database policies.
- Prefer sharing personal contact details through RSVP handling instead of hardcoding phone numbers in public HTML.
- If any credential was shared accidentally, rotate it immediately in the provider dashboard.

## One-time GitHub setup

1. Open your repository: `https://github.com/nusRying/Testamonial`
2. Go to `Settings` -> `Pages`
3. Under `Build and deployment`, set `Source` to `GitHub Actions`

## Deploy updates

Every push to `main` deploys automatically.

```bash
git add .
git commit -m "Set up GitHub Pages deployment"
git push origin main
```

Then watch deployment in the `Actions` tab. When it finishes, open the URL above.

## RSVP delivery and testing

RSVP submissions are sent and stored through Formspree via the RSVP form `action` URL in `index.html`:

`action="https://formspree.io/f/YOUR_FORM_ID"`

Quick setup with Formspree:

1. Open `https://formspree.io/` and create a form.
2. Copy your endpoint URL (format: `https://formspree.io/f/xxxxabcd`).
3. Update the RSVP form `action` in `index.html` with your endpoint URL.
4. Push to `main`.
5. Submit the RSVP form on your live site.
6. Confirm submissions in your Formspree dashboard.

Notes:
- You can also use `data-endpoint="https://formspree.io/f/xxxxabcd"` on the form; this overrides `action`.
- Do not use the placeholder endpoint in production.

## Blessings Wall for everyone (shared live data)

To make every new blessing visible to all visitors, use Supabase for the Blessings Wall shared storage.

Privacy warning:
- This mode is public-by-design for frontend apps. Anyone who can inspect your site source can find the public endpoint and write/read data according to your RLS policies.
- For private family data, keep `data-sync-url=""` and `data-sync-key=""` in `index.html` and use RSVP/Formspree only, or move to a server-side authenticated backend.

1. Create a project at `https://supabase.com/`.
2. In Supabase SQL editor, run:

```sql
create table if not exists public.blessings_wall (
  id bigint generated by default as identity primary key,
  name text not null check (char_length(trim(name)) between 2 and 50),
  message text not null check (char_length(trim(message)) between 8 and 180),
  created_at timestamptz not null default now()
);

alter table public.blessings_wall enable row level security;

create policy "Anyone can read blessings"
on public.blessings_wall
for select
to anon
using (true);

create policy "Anyone can insert blessings"
on public.blessings_wall
for insert
to anon
with check (true);
```

3. In Supabase project settings, copy:
- Project URL (example: `https://abcdxyz.supabase.co`)
- `anon` public key (never use service role key in frontend)

4. In `index.html`, on the Blessings form (`id="blessingForm"`), set:

```html
data-sync-url="https://YOUR_PROJECT_REF.supabase.co"
data-sync-key="YOUR_SUPABASE_ANON_KEY"
data-sync-table="blessings_wall"
```

5. Push changes and redeploy to GitHub Pages.

How it works now:
- Blessings load from shared Supabase table.
- New blessings are inserted into shared table.
- Other visitors get updates automatically (polling every ~20 seconds).
- If shared sync values are blank, the site falls back to Formspree/local mode.

## Guest Gallery Uploads (live for everyone)

This enables guests to upload ceremony photos that appear in the Gallery section for all visitors.

1. In Supabase SQL editor, run:

```sql
create table if not exists public.gallery_photos (
  id bigint generated by default as identity primary key,
  image_url text not null,
  thumb_url text,
  caption text not null check (char_length(trim(caption)) between 3 and 80),
  uploaded_by text,
  created_at timestamptz not null default now()
);

alter table public.gallery_photos enable row level security;

drop policy if exists "Anyone can read gallery photos" on public.gallery_photos;
drop policy if exists "Anyone can insert gallery photos" on public.gallery_photos;

create policy "Anyone can read gallery photos"
on public.gallery_photos
for select
to anon
using (true);

create policy "Anyone can insert gallery photos"
on public.gallery_photos
for insert
to anon
with check (true);
```

2. In Supabase Storage, create a public bucket named `wedding-gallery`, or run:

```sql
insert into storage.buckets (id, name, public)
values ('wedding-gallery', 'wedding-gallery', true)
on conflict (id) do update set public = true;

drop policy if exists "Public can view wedding gallery objects" on storage.objects;
drop policy if exists "Public can upload wedding gallery objects" on storage.objects;

create policy "Public can view wedding gallery objects"
on storage.objects
for select
to anon
using (bucket_id = 'wedding-gallery');

create policy "Public can upload wedding gallery objects"
on storage.objects
for insert
to anon
with check (bucket_id = 'wedding-gallery');
```

3. In `index.html`, update `#galleryUploadForm` data attributes:

```html
data-sync-url="https://YOUR_PROJECT_REF.supabase.co"
data-sync-key="YOUR_SUPABASE_ANON_KEY"
data-gallery-table="gallery_photos"
data-gallery-bucket="wedding-gallery"
```

4. Push to GitHub Pages, then test by uploading from two devices.

Notes:
- Uploaded photos are public to anyone who can access your invitation URL.
- Keep file size limited (default max is 8 MB per image).

## Updating photos and video

You can replace all visible media directly from the project folder.

### Gallery photos

1. Copy your new photo files into the project root (same place as `index.html`), or into a new folder like `images/`.
2. Open `index.html`.
3. Go to the Gallery section and update each image pair:
- `data-full="..."` for the lightbox full-size photo
- `src="..."` for the thumbnail shown on the page
4. Also update `data-caption` and `alt` text for each photo.

Example:

```html
<button class="gallery-item reveal" data-full="images/photo1.jpg" data-caption="Nikkah day" type="button">
  <img src="images/photo1.jpg" alt="Wajahat and Zainab on nikkah day" loading="lazy" decoding="async" />
</button>
```

### Hero video (couple moving forward)

1. Replace the file `Couple_Moving_Forward_On_Road.mp4` with your final video (keep the same name), or update the filename in `index.html`.
2. In `index.html`, find:

```html
<source src="Couple_Moving_Forward_On_Road.mp4" type="video/mp4" />
```

3. If you rename the file, update this `src` value.
